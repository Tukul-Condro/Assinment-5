Important FAQ

What is the Document Object Model (DOM)?

The DOM allows web developers to programmatically access, manipulate, and update the structure and content of a webpage. It acts as the bridge between web documents and scripting languages like JavaScript.

How do you select an element from the DOM?

To select an element from the DOM in JavaScript, you can use several methods provided by the DOM API, depending on the type of element or elements you want to select.
1.getElementById('id'): Selects one element by ID.
2.getElementsByClassName('className'): Selects all elements with the specified class (live collection).
3.getElementsByTagName('tagName'): Selects all elements with the specified tag (live collection).
4.querySelector('selector'): Selects the first element matching a CSS selector.
5.querySelectorAll('selector'): Selects all elements matching a CSS selector (static NodeList).
Each method has its own strengths, so you can choose one based on the type of selection you need to perform.

What is event delegation in the context of the DOM, and why is it useful?

Efficiency: Reduces the need for multiple event listeners.
Scalability: Works seamlessly with dynamically added elements.
Maintainability: Simplifies code by centralizing event handling logic.
Event delegation is a powerful technique for improving performance and reducing complexity when handling events in a DOM with many elements or dynamic content.

How do you manipulate an element's attributes and styles using the DOM?

Attributes can be accessed and modified using methods like getAttribute(), setAttribute(), and removeAttribute(), or by directly modifying properties like element.src or element.id.
Styles can be directly modified using the style property (e.g., element.style.color) or managed through CSS classes using classList methods for better separation and maintainability.
